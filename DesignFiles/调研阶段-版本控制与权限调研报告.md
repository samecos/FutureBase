## 1. 执行摘要

### 1.1 调研目标

本报告针对半自动化建筑设计平台的版本控制与权限管理需求进行技术调研，为平台架构设计提供技术可行性依据。

### 1.2 核心需求

| 需求类别 | 具体需求 | 优先级 |
|---------|---------|--------|
| 版本控制 | 撤销重做功能，历史数据存储到后端 | 高 |
| 权限隔离 | 各工程师账号管理，不能跨账号访问 | 高 |
| 设计迭代 | 支持体块调整后的版本对比 | 高 |
| 多专业协作 | 多专业协作的权限管理 | 高 |

### 1.3 调研结论

基于调研分析，推荐采用以下技术方案：

- **撤销重做**: 命令模式(Command Pattern) + 操作日志存储
- **版本控制**: 增量存储 + 快照策略
- **权限管理**: RBAC模型 + 多租户数据隔离
- **审计追踪**: 结构化审计日志 + 字段级变更追踪
- **身份认证**: JWT + OAuth2.0 双因素认证

### 2.1 Git版本控制原理和架构

#### 2.1.1 Git核心架构

#### 2.1.2 Git数据模型

Git采用**内容寻址存储系统**，核心对象包括：

| 对象类型 | 说明 | 存储方式 |
|---------|------|---------|
| Blob | 文件内容 | SHA-1哈希标识 |
| Tree | 目录结构 | 指向Blob和其他Tree |
| Commit | 提交快照 | 包含作者、时间、父提交指针 |
| Branch | 分支指针 | 轻量级引用 |

#### 2.1.3 对建筑设计平台的启示

- **快照存储**: 借鉴Git的快照机制，设计文件的每次修改存储增量变化
- **分支管理**: 支持设计分支（如方案A、方案B）并行开发
- **三路合并**: 处理多专业协作时的冲突合并

### 2.2 数据库版本控制工具

#### 2.2.1 Liquibase vs Flyway 对比

| 特性维度 | Liquibase | Flyway |
|---------|-----------|--------|
| **核心理念** | "声明式变更" | "SQL即代码" |
| **脚本格式** | XML/YAML/JSON/SQL | 纯SQL为主 |
| **学习成本** | 中等，需理解DSL | 低，DBA友好 |
| **跨数据库兼容** | 强（自动转换语法） | 弱（依赖SQL方言） |
| **回滚能力** | 原生支持多级回滚 | 有限（需商业版） |
| **Diff功能** | 内置Schema差异生成 | 需插件支持 |
| **审计记录** | 多字段+可扩展元数据 | 基础版本记录 |
| **适用场景** | 企业级复杂系统 | 快速迭代的敏捷项目 |

#### 2.2.2 选型建议

对于建筑设计平台：
- **推荐Liquibase**: 需要严格的审计追踪和回滚能力
- **变更集管理**: 每个设计变更作为一个ChangeSet
- **多环境支持**: 开发/测试/生产环境配置分离

### 3.3 撤销栈和重做栈管理

#### 3.3.1 双栈架构

```
┌─────────────────────────────────────────────────────────────┐
│                  撤销/重做栈管理架构                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              HistoryManager (历史管理器)              │  │
│   ├─────────────────────────────────────────────────────┤  │
│   │                                                     │  │
│   │   ┌──────────────┐        ┌──────────────┐         │  │
│   │   │  Undo Stack  │        │  Redo Stack  │         │  │
│   │   │  (撤销栈)     │        │  (重做栈)     │         │  │
│   │   │              │        │              │         │  │
│   │   │  [Cmd_N]     │        │              │         │  │
│   │   │  [Cmd_N-1]   │        │              │         │  │
│   │   │  [Cmd_N-2]   │   ←──→ │  [Cmd_N+1]   │         │  │
│   │   │  [...]       │        │  [Cmd_N+2]   │         │  │
│   │   │  [Cmd_1]     │        │              │         │  │
│   │   └──────────────┘        └──────────────┘         │  │
│   │                                                     │  │
│   │   操作规则:                                          │  │
│   │   1. 执行新命令 → Push到UndoStack, 清空RedoStack   │  │
│   │   2. 撤销操作   → Pop UndoStack, Push到RedoStack   │  │
│   │   3. 重做操作   → Pop RedoStack, Push到UndoStack   │  │
│   │                                                     │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3.3.2 栈管理实现

```typescript
class HistoryManager {
  private undoStack: CommandGroup[] = [];
  private redoStack: CommandGroup[] = [];
  private currentIndex: number = -1;
  private maxHistorySize: number = 1000;  // 最大历史记录数
  private debounceTime: number = 200;     // 防抖时间(合并连续操作)
  
  // 执行命令
  execute(command: DesignCommand): void {
    // 如果在历史中间执行新操作，丢弃后续历史
    if (this.currentIndex < this.undoStack.length - 1) {
      this.undoStack = this.undoStack.slice(0, this.currentIndex + 1);
      this.redoStack = [];
    }
    
    // 检查是否需要合并到当前命令组
    const lastGroup = this.undoStack[this.undoStack.length - 1];
    if (lastGroup && this.shouldMerge(command, lastGroup)) {
      lastGroup.addCommand(command);
    } else {
      const newGroup = new CommandGroup([command]);
      this.undoStack.push(newGroup);
      this.currentIndex++;
    }
    
    // 限制历史记录大小
    this.trimHistory();
    
    // 执行命令
    command.execute();
    
    // 持久化到后端
    this.persistCommand(command);
  }
  
  // 撤销操作
  undo(): boolean {
    if (this.currentIndex < 0) return false;
    
    const commandGroup = this.undoStack[this.currentIndex];
    commandGroup.undo();  // 从后往前执行undo
    
    this.redoStack.push(commandGroup);
    this.currentIndex--;
    
    return true;
  }
  
  // 重做操作
  redo(): boolean {
    if (this.redoStack.length === 0) return false;
    
    const commandGroup = this.redoStack.pop()!;
    commandGroup.redo();  // 从前往后执行redo
    
    this.currentIndex++;
    this.undoStack[this.currentIndex] = commandGroup;
    
    return true;
  }
  
  // 限制历史记录大小
  private trimHistory(): void {
    if (this.undoStack.length > this.maxHistorySize) {
      const overflow = this.undoStack.length - this.maxHistorySize;
      this.undoStack.splice(0, overflow);
      this.currentIndex -= overflow;
    }
  }
}
```

### 3.4 批量操作的撤销处理

#### 3.4.1 命令组设计

```typescript
// 命令组 - 用于批量操作
class CommandGroup implements DesignCommand {
  private commands: DesignCommand[] = [];
  private timestamp: number;
  private groupId: string;
  
  constructor(commands: DesignCommand[] = []) {
    this.commands = commands;
    this.timestamp = Date.now();
    this.groupId = generateUUID();
  }
  
  addCommand(command: DesignCommand): void {
    this.commands.push(command);
  }
  
  execute(): void {
    for (const cmd of this.commands) {
      cmd.execute();
    }
  }
  
  undo(): void {
    // 批量撤销：从后往前执行
    for (let i = this.commands.length - 1; i >= 0; i--) {
      this.commands[i].undo();
    }
  }
  
  redo(): void {
    // 批量重做：从前往后执行
    for (const cmd of this.commands) {
      cmd.redo();
    }
  }
}

// 使用示例：批量移动墙体
class BatchMoveCommand extends CommandGroup {
  constructor(walls: Wall[], offset: Point) {
    const commands = walls.map(wall => 
      new MoveWallCommand(wall.id, offset)
    );
    super(commands);
  }
}
```

## 4. 历史数据管理

### 4.1 全量存储 vs 增量存储

#### 4.1.1 存储策略对比

| 策略 | 描述 | 优点 | 缺点 | 适用场景 |
|-----|------|-----|------|---------|
| **全量存储** | 每次保存完整设计状态 | 实现简单，恢复快 | 存储空间大，传输慢 | 小型设计文件 |
| **增量存储** | 仅保存变化部分 | 存储高效，传输快 | 恢复需依赖历史链 | 大型设计文件 |
| **混合存储** | 全量快照+增量变更 | 平衡效率和存储 | 实现复杂 | 推荐方案 |

#### 4.1.2 推荐方案：混合存储

```
┌─────────────────────────────────────────────────────────────┐
│                  混合存储策略                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   时间线:                                                   │
│                                                             │
│   T0 ──→ T1 ──→ T2 ──→ T3 ──→ T4 ──→ T5 ──→ T6 ──→ T7    │
│   │      │      │      │      │      │      │      │       │
│   ↓      ↓      ↓      ↓      ↓      ↓      ↓      ↓       │
│  全量   增量   增量   增量   全量   增量   增量   增量      │
│  快照   变更   变更   变更   快照   变更   变更   变更      │

│   规则:                                                     │
│   • 每N个增量变更后创建全量快照                             │
│   • 手动保存时创建全量快照                                  │
│   • 关键里程碑创建全量快照                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 历史数据压缩算法

#### 4.3.1 压缩策略

| 压缩层级 | 算法 | 适用数据 | 压缩率 |
|---------|------|---------|--------|
| **传输压缩** | Gzip/Brotli | 网络传输数据 | 60-80% |
| **存储压缩** | LZ4/Snappy | 热数据存储 | 40-60% |
| **归档压缩** | Zstandard | 冷数据归档 | 70-90% |
| **结构压缩** | Delta Encoding | 增量数据 | 80-95% |

#### 4.3.2 增量数据压缩

```typescript
// Delta Encoding 实现
class DeltaCompressor {
  // 计算两个设计状态的差异
  computeDelta(oldState: DesignState, newState: DesignState): Delta {
    const delta: Delta = {
      added: [],
      removed: [],
      modified: []
    };
    
    // 使用Map加速查找
    const oldMap = new Map(oldState.entities.map(e => [e.id, e]));
    const newMap = new Map(newState.entities.map(e => [e.id, e]));
    
    // 找出新增和修改的实体
    for (const [id, newEntity] of newMap) {
      const oldEntity = oldMap.get(id);
      if (!oldEntity) {
        delta.added.push(newEntity);
      } else if (!this.isEqual(oldEntity, newEntity)) {
        delta.modified.push({
          id,
          changes: this.computeEntityDelta(oldEntity, newEntity)
        });
      }
    }
    
    // 找出删除的实体
    for (const [id, oldEntity] of oldMap) {
      if (!newMap.has(id)) {
        delta.removed.push(id);
      }
    }
    
    return delta;
  }
  
  // 应用差异恢复状态
  applyDelta(baseState: DesignState, delta: Delta): DesignState {
    const state = cloneDeep(baseState);
    
    // 应用删除
    state.entities = state.entities.filter(
      e => !delta.removed.includes(e.id)
    );
    
    // 应用修改
    for (const mod of delta.modified) {
      const entity = state.entities.find(e => e.id === mod.id);
      if (entity) {
        Object.assign(entity, mod.changes);
      }
    }
    
    // 应用新增
    state.entities.push(...delta.added);
    
    return state;
  }
}
```

```typescript
// 使用Javers实现数据变更追踪
import { Javers } from 'javers-core';

class DataChangeTracker {
  private javers: Javers;
  
  constructor() {
    this.javers = JaversBuilder.javers()
      .registerValue(ObjectId)
      .build();
  }
  
  // 追踪对象变更
  trackChanges<T>(
    oldObject: T,
    newObject: T,
    context: AuditContext
  ): ChangeLog {
    const diff = this.javers.compare(oldObject, newObject);
    
    const changes: FieldChange[] = diff.changes.map(change => ({
      fieldName: change.propertyName,
      fieldType: change.propertyType,
      oldValue: change.left,
      newValue: change.right,
      changeType: this.mapChangeType(change.type)
    }));
    
    return {
      entityType: oldObject.constructor.name,
      entityId: (oldObject as any).id,
      changes,
      changeCount: changes.length,
      auditedAt: new Date(),
      auditedBy: context.userId
    };
  }
  
  // AOP切面自动记录
  @Audit(operationType = OperationType.UPDATE)
  async updateEntity<T>(
    entityType: string,
    entityId: string,
    updateData: Partial<T>
  ): Promise<T> {
    // 1. 获取旧数据
    const oldEntity = await this.repository.findById(entityId);
    
    // 2. 执行更新
    const newEntity = await this.repository.update(entityId, updateData);
    
    // 3. 变更追踪(由AOP自动完成)
    return newEntity;
  }
}
```

```typescript
// JWT Token 结构
interface JWTPayload {
  sub: string;                   // 用户ID
  iss: string;                   // 签发者
  aud: string;                   // 受众
  iat: number;                   // 签发时间
  exp: number;                   // 过期时间
  jti: string;                   // Token唯一标识
  
  // 自定义声明
  tenantId: string;              // 租户ID
  roles: string[];               // 角色列表
  permissions: string[];         // 权限列表
}

// Token服务
class TokenService {
  private accessTokenExpiry = 15 * 60;      // 15分钟
  private refreshTokenExpiry = 7 * 24 * 60 * 60; // 7天
  
  // 生成Token对
  async generateTokenPair(user: User): Promise<TokenPair> {
    const accessToken = this.generateAccessToken(user);
    const refreshToken = this.generateRefreshToken(user);
    
    // 存储Refresh Token
    await this.refreshTokenRepo.save({
      token: refreshToken,
      userId: user.id,
      expiresAt: new Date(Date.now() + this.refreshTokenExpiry * 1000)
    });
    
    return { accessToken, refreshToken };
  }
  
  // 生成Access Token
  private generateAccessToken(user: User): string {
    const payload: JWTPayload = {
      sub: user.id,
      iss: 'arch-design-platform',
      aud: 'arch-design-api',
      iat: Math.floor(Date.now() / 1000),
      exp: Math.floor(Date.now() / 1000) + this.accessTokenExpiry,
      jti: generateUUID(),
      tenantId: user.tenantId,
      roles: user.roles,
      permissions: user.permissions
    };
    
    return jwt.sign(payload, JWT_SECRET, { algorithm: 'RS256' });
  }
  
  // 验证Token
  async verifyToken(token: string): Promise<JWTPayload> {
    try {
      const payload = jwt.verify(token, JWT_PUBLIC_KEY, {
        algorithms: ['RS256']
      }) as JWTPayload;
      
      // 检查Token是否在黑名单
      const isBlacklisted = await this.tokenBlacklistRepo.exists(payload.jti);
      if (isBlacklisted) {
        throw new Error('Token has been revoked');
      }
      
      return payload;
    } catch (error) {
      throw new UnauthorizedException('Invalid token');
    }
  }
}
```

```typescript
class SessionManager {
  private readonly SESSION_TIMEOUT = 30 * 60; // 30分钟
  private readonly MAX_CONCURRENT_SESSIONS = 5;
  
  constructor(private redis: RedisClient) {}
  
  // 创建会话
  async createSession(
    userId: string,
    context: SessionContext
  ): Promise<Session> {
    // 检查并发会话数
    const existingSessions = await this.getUserSessions(userId);
    if (existingSessions.length >= this.MAX_CONCURRENT_SESSIONS) {
      // 踢掉最早的会话
      await this.revokeSession(existingSessions[0].id);
    }
    
    const session: Session = {
      id: generateUUID(),
      userId,
      tenantId: context.tenantId,
      loginTime: new Date(),
      lastActivity: new Date(),
      ipAddress: context.ipAddress,
      userAgent: context.userAgent,
      activeTokens: []
    };
    
    // 存储会话
    await this.redis.setex(
      `session:${session.id}`,
      this.SESSION_TIMEOUT,
      JSON.stringify(session)
    );
    
    // 添加到用户会话列表
    await this.redis.sadd(`user_sessions:${userId}`, session.id);
    
    return session;
  }
  
  // 刷新会话活动
  async refreshActivity(sessionId: string): Promise<void> {
    const sessionData = await this.redis.get(`session:${sessionId}`);
    if (!sessionData) throw new Error('Session not found');
    
    const session: Session = JSON.parse(sessionData);
    session.lastActivity = new Date();
    
    await this.redis.setex(
      `session:${sessionId}`,
      this.SESSION_TIMEOUT,
      JSON.stringify(session)
    );
  }
  
  // 撤销会话
  async revokeSession(sessionId: string): Promise<void> {
    const sessionData = await this.redis.get(`session:${sessionId}`);
    if (!sessionData) return;
    
    const session: Session = JSON.parse(sessionData);
    
    // 将所有Token加入黑名单
    for (const tokenId of session.activeTokens) {
      await this.tokenBlacklistRepo.add(tokenId);
    }
    
    // 删除会话
    await this.redis.del(`session:${sessionId}`);
    await this.redis.srem(`user_sessions:${session.userId}`, sessionId);
  }
}
```

```typescript
    return { key: rawKey, apiKey };
  }
  
  // 验证API Key
  async validateAPIKey(key: string): Promise<APIKey | null> {
    // 解析Key ID
    const keyId = this.extractKeyId(key);
    
    // 获取存储的Key
    const apiKey = await this.apiKeyRepo.findById(keyId);
    if (!apiKey || apiKey.status !== 'ACTIVE') return null;
    
    // 检查过期
    if (apiKey.expiresAt && apiKey.expiresAt < new Date()) {
      await this.apiKeyRepo.updateStatus(keyId, 'EXPIRED');
      return null;
    }
    
    // 验证Key
    const isValid = await bcrypt.compare(key, apiKey.key);
    if (!isValid) return null;
    
    // 更新最后使用时间
    await this.apiKeyRepo.updateLastUsed(keyId);
    
    return apiKey;
  }
```

```typescript
// 加密服务
class EncryptionService {
  private masterKey: Buffer;
  private algorithm = 'aes-256-gcm';
  
  constructor(private kmsService: KMSService) {
    // 从KMS获取主密钥
    this.masterKey = kmsService.getDataKey();
  }
  
  // 加密数据
  encrypt(plaintext: string): EncryptedData {
    // 生成随机IV
    const iv = crypto.randomBytes(16);
    
    // 生成随机认证标签
    const authTagLength = 16;
    
    // 创建加密器
    const cipher = crypto.createCipheriv(
      this.algorithm,
      this.masterKey,
      iv,
      { authTagLength }
    );
    
    // 加密
    let encrypted = cipher.update(plaintext, 'utf8', 'hex');
    encrypted += cipher.final('hex');
    
    // 获取认证标签
    const authTag = cipher.getAuthTag();
    
    return {
      ciphertext: encrypted,
      iv: iv.toString('hex'),
      authTag: authTag.toString('hex'),
      algorithm: this.algorithm
    };
  }
  
  // 解密数据
  decrypt(encryptedData: EncryptedData): string {
    const decipher = crypto.createDecipheriv(
      this.algorithm,
      this.masterKey,
      Buffer.from(encryptedData.iv, 'hex'),
      { authTagLength: 16 }
    );
    
    decipher.setAuthTag(Buffer.from(encryptedData.authTag, 'hex'));
    
    let decrypted = decipher.update(encryptedData.ciphertext, 'hex', 'utf8');
    decrypted += decipher.final('utf8');
    
    return decrypted;
  }
}

// 敏感字段加密装饰器
function SensitiveField() {
  return function(target: any, propertyKey: string) {
    const getter = function() {
      const encryptedValue = this[`_${propertyKey}`];
      if (!encryptedValue) return null;
      return encryptionService.decrypt(encryptedValue);
    };
    
    const setter = function(value: string) {
      if (value) {
        this[`_${propertyKey}`] = encryptionService.encrypt(value);
      }
    };
    
    Object.defineProperty(target, propertyKey, {
      get: getter,
      set: setter,
      enumerable: true,
      configurable: true
    });
  };
}
```

## 10. 总结与建议

### 10.1 核心结论

1. **版本控制方案**: 推荐采用**命令模式+增量存储**的方案，支持精确的撤销重做，同时保证存储效率。

2. **权限管理方案**: 采用**RBAC+多租户隔离**的方案，通过tenant_id实现数据隔离，通过角色权限实现细粒度控制。

3. **审计追踪方案**: 采用**AOP+Javers**的方案，实现无侵入的操作日志记录和字段级变更追踪。

4. **安全认证方案**: 采用**OAuth2.0+JWT**的方案，支持现代化的身份认证和会话管理。

### 10.2 实施建议

| 阶段 | 任务 | 优先级 | 预计工期 |
|-----|------|-------|---------|
| 第一阶段 | 基础权限框架搭建 | 高 | 2周 |
| 第二阶段 | 撤销重做功能实现 | 高 | 3周 |
| 第三阶段 | 版本存储优化 | 中 | 2周 |
| 第四阶段 | 审计日志系统 | 中 | 2周 |
| 第五阶段 | 安全加固 | 高 | 2周 |

### 10.3 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 历史数据膨胀 | 存储成本高 | 增量存储+定期归档 |
| 权限检查性能 | 响应延迟 | 权限缓存+异步校验 |
| 审计日志性能 | 写入瓶颈 | 异步写入+批量提交 |
| Token安全 | 凭证泄露 | 短有效期+Token轮换 |
