### 1.2 限界上下文定义

| 限界上下文 | 英文名称 | 职责范围 | 核心业务 |
|-----------|---------|---------|---------|
| **草图设计上下文** | SketchContext | 2D/3D草图绘制、几何约束、空间关系 | 快速概念设计、方案推敲 |
| **体块建模上下文** | ElementContext | 建筑元素创建、编辑、几何计算 | 墙体、楼板、柱梁等实体建模 |
| **属性管理上下文** | PropertyContext | 属性定义、赋值、校验、继承 | 100+项专业属性的全生命周期管理 |
| **自动化深化上下文** | AutomationContext | 脚本执行、规则引擎、批量处理 | 半自动化设计深化、批量修改 |
| **脚本引擎上下文** | ScriptContext | 脚本编写、调试、版本、执行 | Python/DSL脚本管理 |
| **版本管理上下文** | VersionContext | 版本控制、分支管理、变更追踪 | 设计迭代历史管理 |
| **数据交换上下文** | DataExchangeContext | IFC导入导出、格式转换、数据同步 | 跨软件协作 |
| **专业协作文本** | CollaborationContext | 多专业并行、冲突检测、协调 | 建筑/结构/机电协同 |

### 1.3 核心域、支撑域、通用域划分

#### 1.3.1 核心域（Core Domain）- 差异化竞争优势

**草图设计域（SketchDomain）**
- **战略重要性**: ★★★★★
- **差异化程度**: 高
- **投资优先级**: 最高
- **核心能力**: 
  - 快速2D草图转3D体块
  - 智能几何约束识别
  - 空间拓扑关系自动建立
  - 住宅标准层模板库

**体块建模域（ElementDomain）**
- **战略重要性**: ★★★★★
- **差异化程度**: 高
- **投资优先级**: 最高
- **核心能力**:
  - 参数化建筑元素（墙体、楼板、柱、梁、洞口）
  - 智能体块关联（依附关系、开洞关系）
  - 几何布尔运算
  - 实时三维可视化

**自动化深化域（AutomationDomain）**
- **战略重要性**: ★★★★★
- **差异化程度**: 极高
- **投资优先级**: 最高
- **核心能力**:
  - 规则驱动的批量属性赋值
  - 脚本化设计深化流程
  - 智能设计检查与修正
  - 效率提升40-50%

#### 1.3.2 支撑域（Supporting Domain）- 必要支撑能力

**属性管理域（PropertyDomain）**
- 建筑/结构/机电三专业100+项属性定义
- 属性继承与覆盖机制
- 属性校验规则引擎
- 属性模板库管理

**脚本引擎域（ScriptDomain）**
- Python脚本运行时环境
- 领域特定语言（DSL）支持
- 脚本版本与依赖管理
- 脚本市场与共享机制

**版本管理域（VersionContext）**
- 设计版本分支管理
- 变更差异可视化
- 版本回滚与合并
- 设计迭代历史追踪

#### 1.3.3 通用域（Generic Domain）- 基础设施能力

**用户权限域（UserDomain）**
- 用户认证与授权
- 角色权限管理（建筑/结构/机电/管理）
- 项目成员管理
- 操作审计日志

**数据交换域（DataExchangeDomain）**
- IFC标准集成
- Revit/DWG格式互导
- 几何数据序列化
- 云端数据同步

**系统配置域（ConfigDomain）**
- 企业标准配置
- 项目模板管理
- 系统参数设置
- 插件扩展管理

#### 2.6.1 脚本示例：标准层墙体属性自动赋值

```python
# 脚本名称: 标准层墙体属性自动赋值
# 脚本类型: PROPERTY_ASSIGNMENT
# 适用专业: 建筑
# 目标类型: ELEMENT (墙体)

def execute(context, parameters):
    """
    为标准层所有墙体自动赋值建筑属性
    """
    # 获取目标墙体
    walls = context.get_elements(
        element_type="WALL",
        floor_type="STANDARD"
    )
    
    for wall in walls:
        # 根据墙体位置判断类型
        if wall.is_external:
            # 外墙属性
            wall.set_property("wall_type", "EXTERNAL")
            wall.set_property("waterproof_grade", "GRADE_2")
            wall.set_property("insulation_type", "EXTERNAL")
            wall.set_property("insulation_material", "XPS挤塑板")
            wall.set_property("insulation_thickness", 80)
            wall.set_property("fire_rating", "GRADE_1")
        else:
            # 内墙属性
            wall.set_property("wall_type", "INTERNAL")
            wall.set_property("fire_rating", "GRADE_2")
            
            # 卫生间墙体特殊处理
            if wall.in_room_type == "BATHROOM":
                wall.set_property("waterproof_grade", "GRADE_2")
                wall.set_property("waterproof_height", 1800)
        
        # 通用属性
        wall.set_property("structure_type", "SOLID")
        wall.set_property("sound_insulation_rating", 45)
    
    return {
        "processed_count": len(walls),
        "message": f"成功处理 {len(walls)} 面墙体"
    }
```

```
│  │      "height": 3000                                              │     │
│  │    },                                                            │     │
│  │    "extrusion_vector": [0, 0, 3000],                             │     │
│  │    "location": [x, y, z],                                        │     │
│  │    "rotation": 0                                                 │     │
│  │  }                                                               │     │

│                                                                          │

│  │  • 单位: 毫米 (mm)                                              │     │
│  │  • 方向: X-东, Y-北, Z-上                                       │     │
│  └────────────────────────────────────────────────────────────────┘     │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 7.1 业务架构核心要点

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       业务架构核心要点总结                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  【核心域聚焦】                                                           │
│  ─────────────────────────────────────────────────────────────────────  │
│  1. 草图设计域: 快速2D草图转3D体块，智能几何约束识别                     │
│  2. 体块建模域: 参数化建筑元素，智能体块关联                             │
│  3. 自动化深化域: 规则驱动的批量属性赋值，效率提升40-50%                 │
│                                                                          │
│  【关键实体设计】                                                         │
│  ─────────────────────────────────────────────────────────────────────  │
│  • Project: 项目实体，管理项目全生命周期                                 │
│  • Floor: 楼层实体，支持标准层复制和参照                                 │
│  • Element: 体块实体，支持10+种建筑元素类型                              │
│  • Property: 属性实体，支持100+项专业属性                                │
│  • Script: 脚本实体，支持Python/DSL脚本                                  │
│                                                                          │
│  【属性继承机制】                                                         │
│  ─────────────────────────────────────────────────────────────────────  │
│  项目级 → 楼层级 → 体块级，三级继承体系                                  │
│  支持属性覆盖和批量操作                                                  │
│                                                                          │
│  【专业协作机制】                                                         │
│  ─────────────────────────────────────────────────────────────────────  │
│  • 建筑/结构/机电三专业并行工作流                                        │
│  • 实时冲突检测和协调机制                                                │
│  • 版本分支管理和变更追踪                                                │
│                                                                          │
│  【数据交换标准】                                                         │
│  ─────────────────────────────────────────────────────────────────────  │
│  • IFC4标准完整支持                                                      │
│  • Revit/ArchiCAD/Tekla等主流BIM软件兼容                                 │
│  • 几何数据多格式支持 (BRep/ExtrudedSolid/CSG)                           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 7.2 实施建议

| 优先级 | 建议项 | 说明 |
|-------|-------|------|
| P0 | 核心域优先开发 | 先实现草图设计、体块建模、自动化深化三个核心域 |
| P0 | 属性系统完善 | 建立完整的100+项属性库和继承机制 |
| P1 | IFC集成 | 实现与Revit等主流软件的IFC数据交换 |
| P1 | 脚本引擎 | 建立Python脚本运行环境和脚本市场 |
| P2 | 专业协作 | 实现三专业并行工作和冲突检测 |
| P2 | 版本管理 | 实现Git-like的版本分支管理 |
| P3 | 云端同步 | 实现多用户实时协作和云端数据同步 |

### 7.3 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| IFC兼容性 | 高 | 充分测试主流BIM软件的IFC导入导出 |
| 属性标准不统一 | 中 | 建立企业级属性标准库，支持自定义扩展 |
| 用户接受度 | 中 | 提供与Revit相似的操作习惯，降低学习成本 |
| 性能瓶颈 | 中 | 采用LOD分级加载，优化大数据量场景性能 |
