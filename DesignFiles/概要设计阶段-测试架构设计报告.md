### 2.2 单元测试规范

#### 命名规范

```
测试类命名: {被测类名}Test
测试方法命名: should_{预期行为}_when_{测试条件}

示例:
- BuildingDesignServiceTest
- should_calculateAreaCorrectly_when_validDimensionsProvided
- should_throwException_when_negativeValueProvided
```

#### 目录结构规范

```
src/
├── main/
│   ├── java/
│   │   └── com/archdesign/
│   │       ├── service/
│   │       │   └── BuildingDesignService.java
│   │       └── ...
│   └── resources/
└── test/
    ├── java/
    │   └── com/archdesign/
    │       ├── service/
    │       │   └── BuildingDesignServiceTest.java      # 单元测试
    │       ├── integration/                           # 集成测试
    │       └── e2e/                                   # E2E测试
    └── resources/
        ├── test-data/                                 # 测试数据
        └── application-test.yml                       # 测试配置
```

#### 测试模板

```java
@DisplayName("建筑设计服务测试")
class BuildingDesignServiceTest {

    @Mock
    private DesignRepository designRepository;
    
    @Mock
    private ValidationService validationService;
    
    @InjectMocks
    private BuildingDesignService buildingDesignService;

    @BeforeEach
    void setUp() {
        MockitoAnnotations.openMocks(this);
    }

    @Nested
    @DisplayName("面积计算测试")
    class AreaCalculationTests {
        
        @Test
        @DisplayName("当提供有效尺寸时，应正确计算面积")
        void should_calculateAreaCorrectly_when_validDimensionsProvided() {
            // Given
            Dimension dimension = new Dimension(10.0, 20.0);
            
            // When
            double area = buildingDesignService.calculateArea(dimension);
            
            // Then
            assertThat(area).isEqualTo(200.0);
        }
        
        @Test
        @DisplayName("当提供负值时，应抛出异常")
        void should_throwException_when_negativeValueProvided() {
            // Given
            Dimension dimension = new Dimension(-10.0, 20.0);
            
            // When & Then
            assertThatThrownBy(() -> buildingDesignService.calculateArea(dimension))
                .isInstanceOf(IllegalArgumentException.class)
                .hasMessageContaining("尺寸不能为负数");
        }
    }
}
```

#### Mock使用规范

| 场景 | Mock工具 | 使用建议 |
|------|---------|---------|
| 服务依赖 | Mockito | 优先使用，隔离被测单元 |
| 静态方法 | Mockito-inline | 谨慎使用，考虑重构 |
| HTTP调用 | WireMock | 模拟外部API响应 |
| 数据库 | @DataJpaTest | 使用内存数据库 |
| 消息队列 | TestContainers | 集成测试使用 |

#### 数据库测试示例

```java
@DataJpaTest
@AutoConfigureTestDatabase(replace = AutoConfigureTestDatabase.Replace.NONE)
@Testcontainers
class DesignRepositoryTest {

    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
        .withDatabaseName("test_db")
        .withUsername("test")
        .withPassword("test");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
    }

    @Autowired
    private DesignRepository designRepository;

    @Test
    @DisplayName("应正确保存和查询设计")
    void should_saveAndFindDesign() {
        // Given
        Design design = Design.builder()
            .name("测试设计")
            .area(100.0)
            .build();

        // When
        Design saved = designRepository.save(design);
        Optional<Design> found = designRepository.findById(saved.getId());

        // Then
        assertThat(found).isPresent();
        assertThat(found.get().getName()).isEqualTo("测试设计");
    }
}
```

@SpringBootTest
@Testcontainers
class DesignEventKafkaTest {

    @Container
    static KafkaContainer kafka = new KafkaContainer(
        DockerImageName.parse("confluentinc/cp-kafka:7.5.0")
    );

    @Autowired
    private DesignEventProducer producer;

    @Autowired
    private DesignEventConsumer consumer;

    @Test
    @DisplayName("应正确发送和接收设计事件")
    void should_sendAndReceiveDesignEvent() throws InterruptedException {

        DesignCreatedEvent event = DesignCreatedEvent.builder()
            .designId("design-001")
            .designName("测试设计")
            .timestamp(Instant.now())
            .build();

        // When
        producer.sendDesignCreatedEvent(event);

        // Then
        await().atMost(Duration.ofSeconds(5))
            .untilAsserted(() -> {
                DesignCreatedEvent received = consumer.getLastEvent();
                assertThat(received).isNotNull();
                assertThat(received.getDesignId()).isEqualTo("design-001");
            });

const errorRate = new Rate('errors');
const responseTime = new Trend('response_time');

// 测试配置
export const options = {
  stages: [
    { duration: '2m', target: 50 },   // 渐进增加
    { duration: '5m', target: 50 },   // 稳定负载
    { duration: '2m', target: 100 },  // 增加负载
    { duration: '5m', target: 100 },  // 稳定负载
    { duration: '2m', target: 200 },  // 峰值负载
    { duration: '5m', target: 200 },  // 稳定峰值
    { duration: '2m', target: 0 },    // 逐渐降低
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95%请求<500ms
    http_req_failed: ['rate<0.01'],    // 错误率<1%
    errors: ['rate<0.05'],             // 自定义错误率<5%
  },
};

// 测试数据
const BASE_URL = __ENV.BASE_URL || 'http://localhost:8080';

export default function () {
  // 设计列表查询
  const listResponse = http.get(`${BASE_URL}/api/v1/designs`, {
    headers: {
      'Authorization': `Bearer ${getAuthToken()}`,
    },
  });
  
  check(listResponse, {
    'list status is 200': (r) => r.status === 200,
    'list response time < 500ms': (r) => r.timings.duration < 500,
  });
  
  errorRate.add(listResponse.status !== 200);
  responseTime.add(listResponse.timings.duration);

  sleep(1);

  // 创建设计
  const createResponse = http.post(`${BASE_URL}/api/v1/designs`,
    JSON.stringify({
      name: `Test Design ${__VU}-${__ITER}`,
      area: 120.5,
      floors: 3,
    }),
    {
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${getAuthToken()}`,
      },
    }
  );
  
  check(createResponse, {
    'create status is 201': (r) => r.status === 201,
    'create response time < 1000ms': (r) => r.timings.duration < 1000,
  });
  
  errorRate.add(createResponse.status !== 201);
  responseTime.add(createResponse.timings.duration);

  sleep(2);
}

| 债务类型 | 识别方式 | 处理策略 | 优先级 |
|---------|---------|---------|-------|
| 缺失的单元测试 | 覆盖率报告 | 逐步补充 | 高 |
| 不稳定的E2E测试 | 失败率统计 | 重构或删除 | 高 |
| 过期的测试数据 | 定期审计 | 更新或清理 | 中 |
| 冗余的测试用例 | 重复度分析 | 合并或删除 | 低 |
