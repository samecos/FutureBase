## 五、设计文档规范要点

### 5.1 架构图规范

- 使用标准图标库（AWS/Azure/阿里云）
- 颜色规范：外部灰、本系统蓝、存储绿、网络橙、安全红
- 线条规范：同步实线、异步虚线
- 布局规范：分层布局（用户→应用→服务→数据）

### 5.2 API文档规范

- 使用OpenAPI 3.0规范
- 统一响应格式
- 完整错误码体系
- JWT认证

### 5.3 命名规范

| 类型 | Java | TypeScript | Python |
|------|------|------------|--------|
| 类/接口 | UserService | IUserService | UserService |
| 方法/函数 | getUserById | getUser | get_user |
| 变量 | userId | userId | user_id |
| 常量 | MAX_SIZE | MAX_SIZE | MAX_SIZE |

### 5.4 版本号规范

采用语义化版本：`MAJOR.MINOR.PATCH`
- MAJOR：不兼容API修改
- MINOR：向下兼容功能新增
- PATCH：向下兼容问题修复
