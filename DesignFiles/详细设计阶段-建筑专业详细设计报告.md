## 1. 业务规则详细设计

### 1.1 草图绘制业务规则

#### 1.1.1 墙体绘制规则

**规则编号：** BR-WALL-001  
**规则名称：** 墙体连续性检查  
**规则描述：** 墙体必须形成闭合或与其他墙体/柱连接，孤立墙体长度不得超过5米  
**触发时机：** 墙体绘制完成、墙体编辑结束  
**校验逻辑：**
```
IF 墙体.端点1.连接状态 == 未连接 AND 墙体.端点2.连接状态 == 未连接 THEN
    IF 墙体.长度 > 5000mm THEN
        返回错误("孤立墙体长度不得超过5米")
    END IF
END IF
```
**错误级别：** 警告  
**自动修复：** 否  

---

**规则编号：** BR-WALL-002  
**规则名称：** 墙体厚度规范  
**规则描述：** 外墙厚度不得小于200mm，内墙厚度不得小于100mm  
**触发时机：** 墙体属性修改、规范检查  
**校验逻辑：**
```
IF 墙体.类型 == "外墙" AND 墙体.厚度 < 200mm THEN
    返回错误("外墙厚度不得小于200mm")
ELSE IF 墙体.类型 == "内墙" AND 墙体.厚度 < 100mm THEN
    返回错误("内墙厚度不得小于100mm")
END IF
```
**错误级别：** 错误  
**自动修复：** 提供一键修复为最小厚度

---

**规则编号：** BR-WALL-003  
**规则名称：** 墙体高度合理性  
**规则描述：** 墙体高度应在楼层高度±10%范围内  
**触发时机：** 墙体高度修改  
**校验逻辑：**
```
标准高度 = 墙体.所属楼层.层高
IF ABS(墙体.高度 - 标准高度) > 标准高度 * 0.1 THEN
    返回警告("墙体高度偏离标准层高超过10%")
END IF
```
**错误级别：** 警告  

#### 1.1.2 楼板绘制规则

**规则编号：** BR-SLAB-001  
**规则名称：** 楼板闭合性检查  
**规则描述：** 楼板轮廓必须是闭合多边形  
**触发时机：** 楼板绘制完成  
**校验逻辑：**
```
IF 楼板.轮廓.起点 != 楼板.轮廓.终点 THEN
    返回错误("楼板轮廓必须是闭合多边形")
END IF
```
**错误级别：** 错误  

---

**规则编号：** BR-SLAB-002  
**规则名称：** 降板区域规则  
**规则描述：** 降板区域必须在楼板范围内，降板深度不得超过300mm  
**触发时机：** 降板区域绘制、降板深度修改  
**校验逻辑：**
```
IF NOT 降板区域.几何.包含于(楼板.轮廓) THEN
    返回错误("降板区域必须在楼板范围内")
END IF
IF 降板区域.深度 > 300mm THEN
    返回错误("降板深度不得超过300mm")
END IF
```
**错误级别：** 错误  

#### 1.1.3 洞口绘制规则

**规则编号：** BR-OPEN-001  
**规则名称：** 洞口位置检查  
**规则描述：** 洞口必须位于墙体或楼板范围内，且距边缘距离不得小于200mm  
**触发时机：** 洞口绘制完成、洞口移动  
**校验逻辑：**
```
承载构件 = 洞口.承载构件
IF 承载构件 == NULL THEN
    返回错误("洞口必须位于墙体或楼板上")
END IF
IF 洞口.边缘距 < 200mm THEN
    返回警告("洞口距边缘距离小于200mm，可能影响结构安全")
END IF
```
**错误级别：** 警告  

---

**规则编号：** BR-OPEN-002  
**规则名称：** 洞口尺寸限制  
**规则描述：** 墙体洞口宽度不得超过墙体长度的80%，楼板洞口面积不得超过楼板面积的30%  
**触发时机：** 洞口尺寸修改  
**校验逻辑：**
```
IF 承载构件.类型 == "墙体" THEN
    IF 洞口.宽度 > 承载构件.长度 * 0.8 THEN
        返回错误("洞口宽度不得超过墙体长度的80%")
    END IF
ELSE IF 承载构件.类型 == "楼板" THEN
    IF 洞口.面积 > 承载构件.面积 * 0.3 THEN
        返回错误("楼板洞口面积不得超过楼板面积的30%")
    END IF
END IF
```
**错误级别：** 错误  

### 1.2 属性定义业务规则

#### 1.2.1 外墙属性规则

**规则编号：** BR-PROP-001  
**规则名称：** 外墙防水等级  
**规则描述：** 屋面接触的外墙必须设置防水措施  
**触发时机：** 外墙属性编辑、屋面绘制完成  
**校验逻辑：**
```
IF 墙体.类型 == "外墙" AND 墙体.接触屋面 == TRUE THEN
    IF 墙体.防水措施 == NULL OR 墙体.防水措施 == "无" THEN
        返回警告("屋面接触的外墙建议设置防水措施")
    END IF
END IF
```
**错误级别：** 警告  

---

**规则编号：** BR-PROP-002  
**规则名称：** 保温材料匹配  
**规则描述：** 外墙保温材料厚度应与节能计算匹配  
**触发时机：** 保温属性修改  
**校验逻辑：**
```
IF 墙体.保温材料.厚度 < 项目.节能要求.最小保温厚度 THEN
    返回警告("保温厚度不满足节能要求")
END IF
```
**错误级别：** 警告  

#### 1.2.2 结构属性规则

**规则编号：** BR-STRUCT-001  
**规则名称：** 结构构件连续性  
**规则描述：** 柱、梁等结构构件应在楼层间保持连续性  
**触发时机：** 结构构件编辑、楼层复制  
**校验逻辑：**
```
FOR EACH 柱 IN 当前楼层.柱列表
    下层柱 = 查找下层对应柱(柱.位置)
    IF 下层柱 == NULL THEN
        返回警告("柱在上下层不连续")
    END IF
END FOR
```
**错误级别：** 警告  

---

**规则编号：** BR-STRUCT-002  
**规则名称：** 梁跨度限制  
**规则描述：** 普通混凝土梁跨度不宜超过12米  
**触发时机：** 梁绘制完成、梁跨度修改  
**校验逻辑：**
```
IF 梁.跨度 > 12000mm THEN
    返回警告("梁跨度超过12米，建议采用预应力或钢结构")
END IF
```
**错误级别：** 警告  

### 1.3 自动化深化规则

**规则编号：** BR-AUTO-001  
**规则名称：** 深化脚本触发条件  
**规则描述：** 当建筑面积、户型布局或结构体系发生变化时，应重新触发深化脚本  
**触发时机：** 模型变更检测  
**校验逻辑：**
```
变更类型 = 检测变更类型()
IF 变更类型 IN ["建筑面积变化", "户型布局变化", "结构体系变化"] THEN
    标记需要重新深化()
    通知相关工程师()
END IF
```

---

**规则编号：** BR-AUTO-002  
**规则名称：** 深化结果校验  
**规则描述：** 深化后的模型应满足基本规范要求  
**触发时机：** 深化脚本执行完成  
**校验逻辑：**
```
深化结果 = 执行深化脚本()
IF 深化结果.构件数量 == 0 THEN
    返回错误("深化结果为空，请检查输入条件")
END IF
IF 深化结果.碰撞数量 > 0 THEN
    返回警告(f"发现{深化结果.碰撞数量}处碰撞，请检查")
END IF
```
**错误级别：** 混合

## 3. 数据校验规则

### 3.1 几何数据校验

| 校验项 | 校验规则 | 错误级别 | 自动修复 |
|--------|----------|----------|----------|
| 墙体闭合性 | 墙体端点必须连接或形成闭合区域 | 错误 | 否 |
| 楼板闭合性 | 楼板轮廓必须是闭合多边形 | 错误 | 否 |
| 构件重叠 | 同类型构件不得重叠 | 错误 | 是（提示合并） |
| 最小尺寸 | 墙体长度≥100mm，楼板面积≥1㎡ | 警告 | 否 |
| 高度一致性 | 同楼层墙体高度差≤50mm | 警告 | 是（统一高度） |
| 垂直对齐 | 上下层墙体中心线偏差≤50mm | 警告 | 否 |

### 3.2 属性数据校验

| 校验项 | 校验规则 | 错误级别 | 自动修复 |
|--------|----------|----------|----------|
| 必填属性 | 类型、材料、厚度为必填项 | 错误 | 否 |
| 数值范围 | 厚度∈[80,500]mm，高度∈[2000,6000]mm | 错误 | 是（设为边界值） |
| 枚举值 | 类型必须在预定义列表中 | 错误 | 否 |
| 专业属性完整 | 结构专业需填写抗震等级、混凝土强度 | 警告 | 否 |
| 属性一致性 | 外墙必须有保温属性 | 警告 | 否 |

### 3.3 专业协作校验

| 校验项 | 校验规则 | 错误级别 | 自动修复 |
|--------|----------|----------|----------|
| 提资完整性 | 建筑专业需向结构专业提资 | 警告 | 否 |
| 确认状态 | 关键构件需相关专业确认 | 警告 | 否 |
| 碰撞检测 | 机电管线与结构构件不得碰撞 | 错误 | 否 |
| 净高检查 | 房间净高需满足规范要求 | 错误 | 否 |

## 6. 实施建议

### 6.1 分阶段实施计划

| 阶段 | 工作内容 | 周期 | 交付物 |
|------|----------|------|--------|
| Phase 1 | 草图绘制功能 | 4周 | 可绘制墙体、楼板、洞口 |

| Phase 4 | 脚本引擎功能 | 4周 | 脚本编辑、执行、调度 |
| Phase 5 | 版本控制功能 | 3周 | 撤销重做、版本管理 |
| Phase 6 | 集成测试优化 | 3周 | 完整系统、性能优化 |

### 6.2 关键成功因素

1. **用户体验优先**：界面操作流畅，符合设计师习惯
2. **性能保障**：大型项目（10000+构件）操作无卡顿
3. **数据安全**：自动保存，防止数据丢失
4. **协作顺畅**：实时同步，冲突友好解决
5. **扩展灵活**：脚本系统支持个性化需求

### 6.3 风险缓解措施

| 风险 | 缓解措施 |
|------|----------|
| 复杂几何运算性能问题 | 采用LOD简化、Web Worker异步计算 |

| 数据丢失风险 | 自动保存、本地缓存、云端备份 |
